Evolución de Estilos: De la Base al Diseño Final

31 de octubre de 2025

En la primera entrada, configuramos la estructura del blog. En esta, vamos a documentar cómo evolucionó nuestra hoja de estilos style.css.

Partimos de un **CSS base, sencillo y elegante**, enfocado puramente en la legibilidad, y luego añadimos tres capas de personalización para darle identidad al blog.

1. El Punto de Partida: CSS Base

El objetivo inicial era tener una lectura cómoda. Esto se logró con:

  • Una fuente serif (como 'Georgia') para el cuerpo del texto (.blog-post), que es más cómoda para lecturas largas.
  • Una fuente sans-serif (como 'Helvetica' o 'Roboto') para los títulos (h1, h2), que ofrece un contraste limpio.
  • Un contenedor (.global-wrapper) con un ancho máximo (max-width: 800px) para centrar el contenido y evitar líneas de texto demasiado largas.

El código CSS base era funcional, pero visualmente muy simple:

body {
  font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Helvetica, Arial, sans-serif;
  background-color: #fdfdfd;
  color: #333;
  line-height: 1.6;
  margin: 0;
  padding: 0;
}

.global-wrapper {
  max-width: 800px;
  margin: 2rem auto;
  padding: 0 1.5rem;
}

.blog-post {
  font-family: 'Georgia', 'Times New Roman', Times, serif;
  font-size: 1.1rem;
}

h1, h2, h3 {
  font-family: 'Helvetica Neue', Helvetica, Arial, sans-serif;
  color: #111;
}

a {
  color: #007acc;
  text-decoration: none;
}

a:hover {
  text-decoration: underline;
}

img {
  max-width: 100%;
  height: auto;
  border-radius: 8px;
}

hr {
  border: 0;
  border-top: 1px solid #eee;
  margin: 2rem 0;
}

footer {
  text-align: center;
  margin-top: 2rem;
  font-size: 0.9rem;
  color: #888;
}

2. Modificación: El Encabezado "Knockout"

Para darle personalidad, modificamos el encabezado (.global-header). En lugar de un simple texto, aplicamos un efecto "knockout" (texto calado):

  1. Se dio al header un fondo gris sólido (#e0e0e0) y un borde.
  2. Al enlace h1 a dentro del header, le asignamos como "fondo" el color del body (#fdfdfd).
  3. Usamos background-clip: text para recortar ese fondo con la forma exacta de las letras.
  4. Finalmente, pusimos color: transparent. Esto hace que las letras sean transparentes, revelando el fondo (color body) que recortamos, creando la ilusión de que el texto está "calado" sobre el fondo gris.
/* ESTILOS DEL HEADER MODIFICADOS */
.global-header {
  /* 1. El relleno gris (sólido) */
  background-color: #e0e0e0; /* Un gris medio */

  /* 2. El rectángulo (borde + padding + esquinas) */
  border: 1px solid #ccc;
  border-radius: 8px;
  padding: 1.5rem 1rem;
  text-align: center;
}

.global-header h1 {
  margin: 0;
}

/* 3. LA MAGIA: LETRAS TRANSPARENTES (KNOCKOUT) */
.global-header h1 a {
  /* Le decimos al enlace que su "fondo"
    sea el mismo que el del body (el blanco/gris claro)
  */
  background-color: #888; 
  
  /* Recortamos ese fondo con la forma exacta del texto */
  -webkit-background-clip: text;
  background-clip: text;
  
  /* Hacemos que el color de la letra sea transparente,
    revelando el "fondo recortado" que acabamos de poner.
  */
  color: transparent;
  
  /* Quitamos el subrayado */
  text-decoration: none;
}

3. Modificación: Barra de Navegación

Añadimos una barra de navegación (.main-nav) simple para separar los enlaces de las entradas. Le dimos un fondo gris muy claro (#f4f4f4) y un borde inferior para separarla visualmente del contenido principal.

Hicimos que los enlaces fueran display: block para que cada uno ocupe su propia línea, facilitando la navegación.

/* 2. ESTILOS DE NAVEGACIÓN */
.main-nav {
  background-color: #f4f4f4;
  padding: 0.5rem 0;
  border-bottom: 1px solid #ddd;
  text-align: left;
  margin-bottom: 1.5rem;
  font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Helvetica, Arial, sans-serif;
}

.main-nav a {
  display: block; /* Pongo cada enlace en su propia línea */
  padding: 0 1rem;
  font-weight: 500;
}

4. Adición Final: Estilos para Código

Siendo un blog técnico, era crucial que los fragmentos de código fueran legibles. Añadimos dos estilos (inspirados en la Entrada 1):

  • Código en línea (<code>): Usando :not(pre) > code, le dimos un fondo gris claro y un color rosado (#c7254e) para resaltarlo dentro de los párrafos.
  • Bloques de código (<pre>): Adoptamos un tema oscuro (estilo "Monokai") con fondo #272822 y texto claro, que es muy popular para la lectura de código.
/* 3. ESTILOS PARA CÓDIGO (¡NUEVO!) */

/* Para código en línea (ej: `npm install`) */
:not(pre) > code {
  font-family: Consolas, 'Courier New', monospace;
  background-color: #f3f4f6; /* Un gris muy claro */
  color: #c7254e; /* Un tono rosado/rojo para código */
  padding: 2px 6px;
  border-radius: 4px;
  font-size: 0.9rem;
}

/* Para bloques de código (logs, comandos multi-línea) */
pre {
  background-color: #272822; /* Un fondo oscuro (estilo Monokai) */
  color: #f8f8f2; /* Texto claro */
  font-family: Consolas, 'Courier New', monospace;
  padding: 1.2rem;
  border-radius: 8px;
  overflow-x: auto; /* Importante para líneas largas */
  font-size: 0.9rem;
  line-height: 1.5;
}

/* Reseteo para el 'code' dentro de 'pre' */
pre code {
    background: none;
    color: inherit;
    padding: 0;
    font-size: inherit;
}

Resultado Final

Con estas modificaciones sobre la base elegante inicial, conseguimos un blog funcional, con una identidad visual clara y una excelente legibilidad para el contenido técnico. ¡Este es el style.css que usamos actualmente!
